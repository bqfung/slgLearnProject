# 第 12 阶段测试清单：存档容错、离线收益与配置校验

## 准备

1. 打开 Unity 工程 `slgLearnProject/`。
2. 点击菜单：`SLG Learn > Build Stage 08 SLG Base Scene`。
3. 点击菜单：`SLG Learn > Clear Stage 08 SLG Save`，清空旧测试存档。
4. 打开生成的场景：`Assets/_Project/Scenes/Stage08_SlgBase.unity`。
5. 点击 Play。

## 存档与离线测试

- 研究一个科技后停止 Play，再次进入 Play，科技研究剩余时间应恢复。
- 研究一个科技后等待一段时间停止 Play，隔一段时间再次进入 Play，研究剩余时间应减少。
- 如果离线时间足够长，重新进入 Play 后科技应完成升级。
- 建筑升级、训练订单、行军订单、资源产出、科技研究应都能在离线结算中推进。
- 出征到据点后停止 Play，再次进入 Play，行军剩余时间应能从存档恢复。
- 如果离线时间足够长，重新进入 Play 后行军应完成并结算战斗。
- 建筑离线升级完成后，剩余离线时间应按升级后的建筑等级继续产出。
- 科技离线研究完成后，剩余离线时间应按新科技加成继续产出。
- 旧版本存档缺少 `strongholds`、`technologies` 或 `marchOrders` 字段时，读取不应报错。
- 旧版本存档读取后，应通过迁移框架升级到当前存档版本。

## 配置校验测试

- 点击菜单：`SLG Learn > Validate All SLG Configs`。
- 正常配置应弹出通过提示。
- Console 中不应出现 Error。
- 校验结果中的 Error/Warning 应包含具体资产路径。
- 点击菜单：`SLG Learn > Config Validator` 应打开独立校验窗口。
- 在校验窗口点击 `Scan All SLG Configs` 后，应显示 Error/Warning 数量。
- 校验窗口中的问题应显示资产路径，点击 `Select` 应在 Project 面板选中对应资产。
- 发起 Build 前应自动执行 SLG 配置校验。
- 如果存在 Error，Build 应被阻止，并在 Console 输出错误原因。
- 如果只存在 Warning，Build 不应被阻止，但 Console 应输出警告。
- 临时把某个资源 id 改错后再次校验，应能看到引用未知资源 id 的 Error。
- 临时把据点推荐战力调成倒序后再次校验，应能看到 Warning。
- 临时把据点首通奖励资源 id 改成不存在的资源 id 后再次校验，应能看到 Error。
- 临时把据点重复奖励资源 id 改成不存在的资源 id 后再次校验，应能看到 Error。
- 临时把据点前置 id 改成不存在的据点 id 后再次校验，应能看到 Error。
- 临时把据点前置 id 改成自己的 id 后再次校验，应能看到 Error。
- 临时把据点前置 id 重复填写两次后再次校验，应能看到 Error。
- 临时把据点驻军兵种 id 改成不存在的兵种 id 后再次校验，应能看到 Error。
- 临时把据点驻军数量改成 0 后再次校验，应能看到 Error。
- 临时把科技效果目标 id 改成不存在的兵种 id 后再次校验，应能看到 Error。
- 临时把科技前置科技 id 改成不存在的 id 后再次校验，应能看到 Error。
- 临时把科技建筑条件等级改到超过建筑最大等级后再次校验，应能看到 Error。

## 当前可接受问题

- 当前校验窗口是基础列表，还没有搜索和过滤。
- 当前存档迁移已有版本入口，但还没有复杂字段改名或拆分迁移案例。
- 当前离线结算已按建筑、训练、科技完成事件分段，但还没有展示详细离线收益报告。

## 通过标准

- 存档读取对旧字段缺失有容错。
- 存档读取会经过统一迁移入口。
- 离线科技研究能推进。
- 离线行军订单能推进并在到达后结算。
- 离线建筑升级和科技完成后的剩余时间会按新状态继续结算。
- `ScriptableObjects` 目录下的 SLG 配置可以一键校验。
- 可以在独立窗口中查看并定位配置错误。
- 构建前会自动阻止 Error 级别配置问题。
- 配置错误能提前在编辑器中被发现。
